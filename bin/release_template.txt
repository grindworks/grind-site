Release title
v1.6.2 - Updater Core Fix & CPT Usability Improvements

Release notes
This patch release addresses a critical bug in the updater core and significantly improves the usability of Custom Post Types (CPT).

### ✨ New Features & Enhancements
- **CPT Quick-Create Button:** Added a "+ New" button to the admin header, allowing users to easily create their first custom post (e.g., Products, Events) even if it's not yet visible in the main sidebar. This resolves a key usability issue for new `theme.json` features.

### 🛡️ Security & Stability
- **Critical Updater Fix:** Fixed a core bug in the updater that incorrectly excluded files from being updated if they had the same name as a file in the root directory (e.g., `admin/config.php`). This ensures all future updates are applied correctly and completely.
- **Improved Code Clarity:** Renamed `admin/config.php` to `admin_menu.php` to prevent future naming conflicts and improve architectural clarity.

---

## 🇯🇵 日本語 (Japanese)

このパッチリリースは、アップデーターのコアに関する重大なバグを修正し、カスタム投稿タイプ（CPT）の使いやすさを大幅に向上させます。

### ✨ 新機能・改善
- **CPTクイック作成ボタン:** 管理画面のヘッダーに「+ New」ボタンを追加しました。これにより、`theme.json`で定義したカスタム投稿タイプ（商品、イベントなど）がまだサイドバーに表示されていなくても、ユーザーは最初の1件を簡単に作成できるようになります。

### 🛡️ セキュリティと安定性
- **重大なアップデーターの修正:** アップデーターが、ルートディレクトリの除外ファイル（例: `config.php`）と同じ名前を持つファイルを誤って更新対象から除外してしまうコアのバグを修正しました。これにより、将来のすべてのアップデートが完全かつ正確に適用されるようになります。
- **コードの明確性の向上:** 将来的な名前の衝突を防ぎ、アーキテクチャの明確性を向上させるため、`admin/config.php` を `admin_menu.php` にリネームしました。
