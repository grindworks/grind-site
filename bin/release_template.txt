Release title
v1.6.4 - Contact Form Settings, Editor UX Polish & Performance Updates

Release notes
This release introduces customizable Contact Form Settings, a God-Tier UX improvement for the block editor, and important performance and security optimizations.

### ✨ Enhancements
- **Contact Form Settings:** You can now fully customize the contact form directly from the admin panel! Set custom recipient emails, subject dropdown options, success messages, and auto-reply email templates. Additionally, automatic `Reply-To` header mapping is now supported, allowing you to reply directly to users from your email client.
- **Editor UX (God-Tier):** Added the ability to paste images directly from your clipboard (e.g., screenshots) into the block editor. Images are automatically uploaded and inserted as blocks.
- **Performance:** Optimized SVG image handling to preload metadata correctly, preventing N+1 query issues and ensuring `alt` tags are properly rendered.

### 🛡️ Security & Stability
- **Security (ReDoS Prevention):** Hardened the regular expressions used in Markdown exports to prevent potential ReDoS (Regular Expression Denial of Service) attacks.
- **Bug Fixes:**
  - Fixed an issue where custom field image previews were not properly restored when recovering from an auto-saved draft.

## 🇯🇵 日本語 (Japanese)

このリリースでは、待望の「お問い合わせフォーム設定」機能の追加に加え、ブロックエディタの執筆体験（UX）を最高峰に引き上げる改善、およびパフォーマンスとセキュリティの最適化が行われています。

### ✨ 新機能・改善
- **お問い合わせフォーム設定の追加:** 管理画面からお問い合わせフォームを柔軟にカスタマイズできるようになりました！専用の受信先メールアドレス、件名の選択肢、送信完了メッセージ、さらにユーザーへの「自動返信メール」の件名・本文を自由に設定可能です。また、管理者宛の通知メールに `Reply-To` が自動設定されるようになり、メールソフトの「返信」ボタンで直接ユーザーへ返信できるようになりました。
- **エディターUXの極み:** クリップボードにコピーした画像（スクリーンショットなど）を直接エディタにペーストできるようになりました。自動でアップロードされ、画像ブロックとして挿入されます。
- **パフォーマンス最適化:** SVG画像のメタデータを適切に事前ロードするよう修正し、N+1クエリを防ぐとともに `alt` 属性が正しく出力されるように改善しました。

### 🛡️ セキュリティとバグ修正
- **セキュリティ (ReDoS対策):** Markdownエクスポート機能で使用される正規表現を強化し、PCREバックトラック制限超過（ReDoS攻撃）のリスクを排除しました。
- **バグ修正:**
  - 自動保存（ドラフト）から記事を復元する際、カスタムフィールドの画像プレビューが正しく復活しないバグを修正しました。
